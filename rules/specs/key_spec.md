# 人机输入规范

> 触发条件：涉及按键、旋钮、拨码开关、触摸按键、外部中断输入、短按/长按、多击、UI 交互或蜂鸣器反馈时，必须先阅读本文件。

本文件不固定按键数量、引脚或交互含义。输入设计必须来自当前硬件、原理图、工程配置和产品交互需求。

## 开发前必须确认

- 输入器件类型：机械按键、编码器、拨码、触摸、霍尔、光电或其他输入。
- 引脚、端口、复用功能、中断线、上拉/下拉、电平有效性和外部电路。
- 是否与其他外设复用或冲突。
- 是否需要短按、长按、连按、组合键、按住重复或释放触发。
- 当前工程已有输入驱动、事件队列、UI 层和蜂鸣器反馈接口。

## 驱动设计原则

- 输入驱动只负责硬件采样、消抖和事件生成，不直接执行业务动作。
- 业务层或 UI 层只消费抽象事件，不直接读取 GPIO。
- 机械输入必须消抖；消抖方式按硬件和实时性选择中断、定时扫描或二者结合。
- 中断路径只记录边沿、时间戳或原始状态，不做复杂业务处理。
- 多个中断或主循环共享状态时，必须保证原子性和可见性。
- 时间判断必须考虑计数器回绕，并使用当前工程已有安全时间比较方式。

## 交互设计原则

- 每个输入事件必须有清晰语义，例如确认、返回、翻页、增减、启停、模式切换。
- 短按和长按不能在同一次操作中重复触发，除非产品交互明确要求。
- 高风险操作应使用长按、二次确认或组合操作，避免误触。
- 参数调整应有范围限制、步进规则、保存策略和取消策略。
- 反馈音、灯光或显示提示只在事件确认后触发，不在中断里直接控制。

## 可移植性要求

- 引脚宏、事件枚举、消抖时间和长按时间必须按当前项目定义，不在规则中固定。
- 跨芯片移植时，应重新确认中断线合并方式、回调参数是否包含端口信息、低功耗唤醒行为和输入滤波能力。
- 如果厂商库或 BSP 已有按键扫描框架，优先适配该框架，不重复造一套并行输入系统。

## 禁止事项

- 禁止固定某组引脚或某种四键布局作为通用规则。
- 禁止在中断回调中切页面、改阈值、发送通信帧或刷新显示。
- 禁止用阻塞等待实现长按或释放检测。
- 禁止在未确认外部电路时假设按下电平。
- 禁止让输入事件在主循环读取时发生竞态丢失。

## 完成检查清单

- 已确认引脚、电平、中断线和外部电路来源。
- 输入驱动、UI 层和业务动作边界清晰。
- 消抖、长按、重复触发和组合操作语义明确。
- 共享状态同步方式可靠。
- 换芯片后需要复核的中断和低功耗行为已经说明。

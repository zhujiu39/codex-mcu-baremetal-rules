# 采样与滤波规范

> 触发条件：涉及传感器采集、ADC 数据处理、测距、环境数据平滑、阈值判断、控制输入或滤波算法时，必须先阅读本文件。

滤波不是固定参数表。算法和参数必须来自信号特性、采样周期、噪声来源、控制响应要求和实测数据。

## 开发前必须确认

- 传感器类型、输出接口、单位、量程、分辨率、转换时间和推荐采样周期。
- 数据用途：显示监测、告警判断、闭环控制、统计记录或通信上报。
- 噪声来源：电源、继电器、电机、泵、通信干扰、机械抖动、安装位置或环境变化。
- 当前工程 ADC、DMA、定时器触发、校准和数据缓存方式。
- 阈值判断是否需要迟滞、防抖、保持时间或异常值剔除。
- 是否已有通用滤波模块，优先复用而不是重复实现。

## 算法选择原则

- 慢变化模拟量可使用指数平滑或滑动平均，但参数必须按响应时间和噪声幅度调整。
- 存在偶发尖峰的数据，应先考虑中值、限幅或异常值剔除，再做平滑。
- 阈值附近反复动作时，优先使用迟滞、保持时间或连续命中计数，不只靠加重滤波。
- 快速闭环控制输入不能过度平滑，必须评估相位延迟和响应时间。
- 数字输入类信号应按事件防抖或状态确认处理，不直接套模拟滤波。
- 测距、流量、转速等带物理模型的数据，应结合量程、变化速度和无效读数规则处理。

## 参数确定原则

- 参数必须能解释来源：数据手册建议、项目实测、控制响应需求或已有工程经验。
- 调参前至少记录一段原始值、滤波值、采样周期和实际工况。
- 评估静态噪声、阶跃响应、异常峰值、阈值误触发和恢复时间。
- 更换芯片、ADC 参考电压、采样电路、供电、安装位置或执行机构后，必须重新评估参数。
- 参数应集中定义或封装，避免散落在多个模块中。

## ADC 与采样链路

- ADC 多通道或高频采样应评估 DMA、硬件触发和采样缓存，不能默认用 CPU 逐点阻塞读取。
- ADC 原始值转换到物理量时，必须明确参考电压、分压比例、校准系数和单位。
- 原始值、校准值、滤波值和状态值应分层保存，不混用含义。
- 超量程、断线、短路、无效码值和转换失败必须显式标记。

## 禁止事项

- 禁止把某个传感器的固定参数当成所有项目默认值。
- 禁止没有采样周期和噪声依据就套用滤波算法。
- 禁止用滤波掩盖传感器离线、线路故障或 ADC 配置错误。
- 禁止让滤波函数静默接受非法输入并输出看似正常的数据。
- 禁止为了显示稳定牺牲安全控制所需的响应速度。

## 完成检查清单

- 已说明数据单位、采样周期、量程和无效值规则。
- 已说明滤波算法选择理由和参数来源。
- 阈值判断已考虑迟滞、防抖或保持时间。
- 采样链路不会阻塞主循环或破坏其他实时任务。
- 更换芯片或硬件后需要复核的参数已经集中可查。
